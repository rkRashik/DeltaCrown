## Rollback Plan

### If deployment fails (build error or health check fails):
1. Render auto-rolls back to the last successful deploy
2. If not: **Render Dashboard â†’ Manual Deploy â†’ select last working commit**

### If lifecycle cron fails:
1. Check cron-job.org logs for the error response
2. Verify `CRON_SECRET` matches between Render env and cron-job.org header
3. Test manually:
   ```bash
   curl -X POST https://deltacrown.xyz/api/lifecycle/cron/ \
     -H "Authorization: Bearer <CRON_SECRET>"
   ```
4. If endpoint itself is broken: fix code, push, redeploy
5. **Temporary fallback:** Set `ENABLE_CELERY_BEAT="1"` in Render env vars (uses more RAM but works without external cron). This is a last resort â€” monitor RAM usage closely.

### If site is up but broken (500 errors, data issues):
1. Check Render logs: **Render Dashboard â†’ Logs**
2. If database migration issue: `python manage.py migrate --check`
3. If Redis connection issue: Check Upstash dashboard
4. If Neon DB timeout: Check Neon dashboard for connection pool status
